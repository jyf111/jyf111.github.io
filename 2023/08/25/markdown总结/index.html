¶代码块

  • 很多 Shell 命令都要粘贴到终端中去执行，因此最好避免在 Shell 命令中使用任何换行操作；可以在行尾使用一个\，这样既能避免命令换行，又能提高源码的可读性。

  • 建议不要在没有输出内容的 Shell 命令前加$。在命令没有输出内容的情况下，$是没有必要的，因为内容全是命令，我们不会把命令和输出的内容混淆。

  • 建议在有输出内容的 Shell 命令前加上$，这样会比较容易区分命令和输出的内容。（$代表普通用户，#代表超级用户）

¶排版 Tips

  1. 安装markdownlint插件。
  2. 使用Trailing Spaces插件。
  3. 建议中文和英文之间加空格，中文/英文和数字之间也要加空格。（试了几个hexo插件都存在一些问题，这条看情况遵守吧）
  4. 英文标点符号（如，.；：？）与后面的字符之间需要加空格，与前面的字符之间不需要加空格。
  5. 中文标点符号和数字、中文、英文之间不需要添加空格。
  6. 中文排版使用全角标点符号，英文排版使用半角标点符号。
  7. 专有名词注意大小写问题。
  8. 括号的用法：在括号内只有英文时用英文括号，并在括号前后加一个半角空格；在括号内有中文时用中文括号。
  9. 英文没有顿号，因此在英文和阿拉伯数字并列时用逗号，否则用顿号。
